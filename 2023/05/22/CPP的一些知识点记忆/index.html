iterator迭代器：

*iter : return reference to the element denoted by iter

iter -> mem : 等价于 (*iter).mem

++ iter : increments iter to refer to the next element in the container

iter ++ : return the copy of the original iterator

== 、!= : equal if both iterators are pointing to the same position

any object that could be traversed using the range-for loop must provide begin() and end(), which return an iterator that supports.

  • const_iterator

​ it here is of type vector<int>::const_iterator

  • other iterator operations

​ iter + n : return an iterator positioned n elements forward

​ iter - n: return an iterator positioned n elements backward

​ iter += n :

​ iter -= n

​ iter1 - iter2 : return the distance between two iterators

​ <, <=, >, >= : 关系比较

the return-type of iter1 - iter2 is a type-alias member named difference_type , 例如： string::difference_type

并不是所有的迭代器都支持 <、 <=、 >、 >= ,但是所有的迭代器都支持 == 和 !=

  • 在一个常量容器中，begin()和end()函数返回 const_iterators. C++11中添加了cbegin() 和cend()函数

迭代器提供五种类型的迭代器： input-iterator,output-iterator,forward-iterator,bidirectional-iterator,random-access-iterator.

std::istream_iterator is one of the category input-iterator

std::ostream_iterator is one of the category output-iterator

forward-iterator支持operator*、 operator++、 operator==、 operator !=

bidirectional-iterator是forward-iterator， 支持operator--

random-access-iterator是bidirectional-iterator，支持it1 - it2, it + n, it - n, n + it, += ,-=, it[n], <, <=, >, >=

  • vector,deque, string and array have random-access-iterators
  • list has bidirectional-iterator
  • forward_list has forward-iterator

stack, queue, priority_queue 是 container adapters (容器适配器)

stack和queue底层是deque, priority_queue底层是vector

  • they are NOT containers and have no iterators
  • they use a container to store data, and re-define the interfaces to rsemble the corresponding data structures.

类型转换：

声明了一个 Rational 对象 r，并用 some_value() 给它赋值。然后，通过将 r 赋值给 double 类型的变量 x，触发了类型转换运算符的调用，将 r 转换为 double 类型的值，并将该值赋给 x。

这段代码的目的是将自定义的有理数类型 Rational 转换为内置的 double 类型，以便在计算和使用浮点数的场景中进行操作。通过重载类型转换运算符，可以使 Rational 对象能够以与 double 类型相似的方式进行使用，例如进行数学运算或赋值给其他变量。

宏定义和const常量的区别：

#define：相当于字符串替换、预处理器处理、无类型安全检查、不分配内存、存储在代码段、可通过#undef取消

const常量： 常量声明、编译器处理、有类型安全检查、要分配内存、存储在数据段、不可取消

inline内联函数：

  • 相当于把内联函数里面的内容写在调用内联函数处
  • 相当于不用执行进入函数的步骤，直接执行函数体
  • 相当于宏，却比宏多了类型检查，真正具有函数特性
  • 编译器一般不内联包含循环、递归、switch等复杂操作的内联函数
  • 在类声明中定义的函数，除了虚函数的其他函数都会自动隐式地当成内联函数

​ 编译器对inline函数的处理步骤：

  • 将inline函数体复制到inline函数调用处
  • 为所用inline函数中的局部变量分配内存空间
  • 将inline函数的输入参数和返回值映射到调用方法的局部变量空间中

inline函数的优点：同宏一样在调用处进行代码展开，省去了参数压栈，栈帧开辟与回收，结果返回等，提高程序运行速度、相比于宏在代码展开时会做安全检查或自动类型转换，在类中声明同时定义的成员函数，自动转化为内联函数，因此内联函数可以访问类的成员变量，宏定义则不能，内联函数在运行时可调试，而宏定义不能。

inline函数的缺点：是否内联取决于编译器，内联函数只是对编译器的建议、inline函数无法随着函数库的升级而升级，inline函数的改变需要重新编译、代码膨胀。

  • 虚函数可以是内联函数，inline可以修饰虚函数，但是当虚函数表现多态性的时候不能内联。

volatile关键字：

volatile关键字是一种类型修饰符，用它声明的类型变量可以被某些编译器未知的因素更改，所以使用volatile告诉编译器不应对这样的对象优化、volatile关键字声明的变量，每次访问时都必须从内存中取值（没有被volatile修饰的变量可能由于编译器优化从CPU取值）、指针可以是volatile。
