CPP的一些知识点记忆

explicit关键字：

  • explicit修饰构造函数时，可以防止隐式转换和复制初始化
  • explicit修饰转换函数时，可以防止隐式转换

隐式转换情况：

  • 数值类型之间的隐式转换：小范围整数向大范围整数类型的转换，如int赋值给long、浮点数类型之间的隐式转换，如float赋值给double

  • 类型提升，将较低精度的整数类型自动提升为较高精度的整数类型，如short赋值给int。

  • 派生类向基类的隐式转换：可以将派生类的对象隐式转换为基类类型，适用于多态，允许基类指针或引用指向派生类对象

  • 用户自定义类型转换：通过重载类型转换运算符或者定义转换函数来自定义类型之间的隐式转换

emplace:

emplace_back()：使用传递给函数的参数直接构造新元素，不需要进行额外的拷贝或移动操作，可以直接将构造元素所需的参数传递给emplace_back函数、接受构造元素所需的参数，并在容器的尾部直接构造新元素，参数会被传递给元素类型的构造函数、由于避免了额外的拷贝或移动操作，开销较小。

类型转换：

声明了一个 Rational 对象 r，并用 some_value() 给它赋值。然后，通过将 r 赋值给 double 类型的变量 x，触发了类型转换运算符的调用，将 r 转换为 double 类型的值，并将该值赋给 x。

这段代码的目的是将自定义的有理数类型 Rational 转换为内置的 double 类型，以便在计算和使用浮点数的场景中进行操作。通过重载类型转换运算符，可以使 Rational 对象能够以与 double 类型相似的方式进行使用，例如进行数学运算或赋值给其他变量。

重载类型转换运算符：

通过重载类型转换运算符来自定义类型之间的转换规则，类型转换运算符以关键字operator开头，后面跟着要转换的目标类型，然后是函数体。

宏定义和const常量的区别：

#define：相当于字符串替换、预处理器处理、无类型安全检查、不分配内存、存储在代码段、可通过#undef取消

const常量： 常量声明、编译器处理、有类型安全检查、要分配内存、存储在数据段、不可取消

inline内联函数：

  • 相当于把内联函数里面的内容写在调用内联函数处
  • 相当于不用执行进入函数的步骤，直接执行函数体
  • 相当于宏，却比宏多了类型检查，真正具有函数特性
  • 编译器一般不内联包含循环、递归、switch等复杂操作的内联函数
  • 在类声明中定义的函数，除了虚函数的其他函数都会自动隐式地当成内联函数

​ 编译器对inline函数的处理步骤：

  • 将inline函数体复制到inline函数调用处
  • 为所用inline函数中的局部变量分配内存空间
  • 将inline函数的输入参数和返回值映射到调用方法的局部变量空间中

inline函数的优点：同宏一样在调用处进行代码展开，省去了参数压栈，栈帧开辟与回收，结果返回等，提高程序运行速度、相比于宏在代码展开时会做安全检查或自动类型转换，在类中声明同时定义的成员函数，自动转化为内联函数，因此内联函数可以访问类的成员变量，宏定义则不能，内联函数在运行时可调试，而宏定义不能。

inline函数的缺点：是否内联取决于编译器，内联函数只是对编译器的建议、inline函数无法随着函数库的升级而升级，inline函数的改变需要重新编译、代码膨胀。

  • 虚函数可以是内联函数，inline可以修饰虚函数，但是当虚函数表现多态性的时候不能内联。

volatile关键字：

volatile关键字是一种类型修饰符，用它声明的类型变量可以被某些编译器未知的因素更改，所以使用volatile告诉编译器不应对这样的对象优化、volatile关键字声明的变量，每次访问时都必须从内存中取值（没有被volatile修饰的变量可能由于编译器优化从CPU取值）、指针可以是volatile。

多态的实现原理：

  • 继承和虚函数机制：在OOP中，通过继承机制可以实现多个子类从同一个父类派生出来，而通过在父类中定义虚函数，在子类中重写虚函数的方式，可让不同类型的对象在同一个接口上表现出不同的行为，在运行时，程序根据对象的实际类型来动态的调用相应的虚函数。
  • 指针和引用：通过指针和引用可以实现不同类型的对象进行统一的处理，由于指针和引用都可以指向不同类型的对象，因此可以将它们作为参数传递给同一个函数，从而实现函数的多态性。在运行时，程序会根据指针或引用所指向的对象的实际类型来动态地调用相应的函数。

如何实现多态：

  • 虚函数表：在含有虚函数的类中，编译器会为该类生成一个虚函数表，该表存储了指向虚函数地址的指针，当通过父类指针或引用调用虚函数时，实际上是通过虚函数表来获取子类的函数地址进行调用。这样做的好处是调用效率高，缺点是需要额外的内存存储虚函数表。
  • 虚函数指针：在含有虚函数的类中，编译器会为该类生成一个虚函数指针，该指针指向虚函数地址，当通过父类指针或引用调用虚函数时，实际上是通过虚函数指针来获取子类的函数地址进行调用。这样做的好处是不需要额外的内存存储虚函数表，但是每个对象需要额外存储一个虚函数指针，会增加内存消耗。

static_assert:

用于在编译期间进行静态断言，用于检查编译时的条件，如果条件为假，则会在编译期间引发编译错误；

接收两个参数：

  • 第一个参数是一个编译时的布尔表达式，用于表示要检查的条件；

  • 第二个参数是一个字符串字面值，用于在条件为假时生成的编译错误的错误信息；

std::enable_if:

用于根据条件来控制模板的有效实例化，通常与函数模板一起使用，以在编译期间进行条件分发；

接收两个参数：

  • 第一个参数是一个布尔值表达式，用于确定条件是否满足；

  • 第二个参数是要返回的类型，如果条件满足，则返回这个类型，否则这个模板将导致编译失败，因为没有对应的类型返回；

std::conditional使用::type成员来获取结果类型

在 Slist_iterator 类中，使用了 std::conditional 模板来根据 is_const 的值选择不同的类型。具体来说，根据 is_const 是否为 true，选择了不同的引用类型和指针类型作为 reference 和 pointer 的别名。

  • 如果 is_const 为 true，则使用 const T & 作为 reference 的类型，并使用 const T * 作为 pointer 的类型。这表示 Slist_iterator 是一个常量迭代器，只能读取但不能修改所指向的元素。
  • 如果 is_const 为 false，则使用 T & 作为 reference 的类型，并使用 T * 作为 pointer 的类型。这表示 Slist_iterator 是一个非常量迭代器，允许读取和修改所指向的元素。

std::distance:

用于计算两个迭代器之间的距离

  • first：指向要计算距离的起始位置的迭代器。
  • last：指向要计算距离的结束位置的迭代器。
  • 返回值为 first 和 last 之间的距离，即迭代器的移动步数。返回值的类型为 typename std::iterator_traits<InputIt>::difference_type，根据迭代器类型的不同，返回值类型也会有所不同。

std::iterator traits:

std::iterator_traits 提供了一组类型别名和静态成员，用于访问迭代器的属性。它的主要目的是为了在泛型算法中，能够通过统一的接口获取迭代器的相关信息，而不依赖于具体的迭代器类型。

  1. iterator_category：迭代器的类型类别，如 std::input_iterator_tag、std::output_iterator_tag、std::forward_iterator_tag、std::bidirectional_iterator_tag、std::random_access_iterator_tag 等。
  2. value_type：迭代器指向的值类型。
  3. difference_type：迭代器之间的距离类型，通常用于表示两个迭代器之间的距离。
  4. pointer：指向迭代器所指向对象的指针类型。
  5. reference：迭代器所指向对象的引用类型。

SFINAE:

  • When substitution failure happens, the compiler tries some other solutions instead of reporting an error.
  • If we can transform an error into a substitution failure…

substitution failure is not an error;在C++中，当进行模板实例化时，编译器会对模板参数进行替换，生成对应的代码，但是在某些情况下，模板实例化可能会失败，例如，对于某些类型无法调用特定的成员函数，或者某些类型无法匹配特定的模板约束条件等。

SFINAE 允许在模板实例化失败时，编译器不报错，而是继续尝试其他可能的模板实例化。这样，我们可以根据模板参数的不同情况，选择不同的模板实例化路径，从而实现编译时的条件分支。

SFINAE 最常见的应用是通过函数重载和模板特化来实现。例如，可以使用 SFINAE 技术编写一个根据类型是否拥有特定成员函数进行重载的函数。

  • const_cast<const Args*>(&args) 展开成 const_cast<const T1 *>(&x1), const_cast<const T2 *>(&x2), const_cast<const T3 *>(&x3)
  • sizeof ...(args)： 计算参数包里的参数个数

模板元编程：

首先，定义了一个模板结构体 AreSame，它接受两个类型参数 X 和 Y。初始的结构体模板中，value 成员被设置为 false，表示默认情况下两个类型不相同。

然后，通过特化（Partial Specialization）的方式，为 AreSame 结构体模板提供了一个偏特化（Partial Specialization）版本。偏特化版本只会在类型参数 X 和 Y 相同时匹配。在偏特化版本中，将 value 成员设置为 true，表示两个类型相同。

这样，在使用 AreSame 结构体时，可以通过访问其静态成员 value 来判断两个类型是否相同。如果 value 为 true，则表示两个类型相同；如果 value 为 false，则表示两个类型不相同。
